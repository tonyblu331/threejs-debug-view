## `DebugViewLayer`

Batteries-included R3F component for the default debug overlay and Leva controls.
Import it from `threejs-debug-view/r3f`.

```tsx
<DebugViewLayer />
```

Use this for quick integration. Use `DebugViews` when your app owns the controls, layout state, custom view list, or viewport assignments.

`DebugViewLayer` mounts the Leva panel on `document.body` by default (outside the R3F tree) so the panel appears without a second import. Set `showLeva={false}` when you want to mount `DebugViewLeva` yourself elsewhere in your app shell.

Important props:

| Prop | Purpose |
| --- | --- |
| `views` | Optional custom view list. Defaults to `DEFAULT_DEBUG_VIEWS`. |
| `viewLabels` | Optional labels passed to the bundled Leva controls. |
| `maxPaneCount` | Upper bound for panes, rows, and columns in the controls. |
| `showEnabledControl` | Shows the Leva `Enabled` toggle. Defaults to `true`. The demo sets this to `false`. |
| `showLeva` | Mounts the bundled Leva control panel on `document.body`. Defaults to `true`. Set to `false` when mounting `DebugViewLeva` yourself. |

## `DebugViewLeva`

Bundled Leva panel for the debug overlay controls. Import it from `threejs-debug-view/r3f`.

`DebugViewLayer` mounts this automatically unless you pass `showLeva={false}`. Mount it yourself when the panel should live in a specific place in your app shell while still sharing the same Leva store as `useDebugViewsControls`. Both `DebugViewLayer` and `DebugViewLeva` portal the panel to `document.body` so Leva never enters the R3F reconciler.

## `DebugViews`

R3F component for presenting debug views inside a React Three Fiber `<Canvas>`.
Import it from `threejs-debug-view/r3f`.

The root package export stays React-free. Use it for core helpers, built-in view definitions, render planning, headless WebGPU runtime, and public types. See [Headless Runtime](/threejs-debug-view/guides/headless-runtime/).

Important props:

| Prop | Purpose |
| --- | --- |
| `views` | Registered debug views. |
| `activeView` | Selected view for single/overlay flows. |
| `layout` | Layout preset or layout config. |
| `paneCount` | Number of visible panes for row, column, and grid layouts. |
| `diagonalAngle` | Angle in degrees for `split-diagonal` and `breakdown`. `breakdown` defaults to `25` degrees. |
| `maxDiagonalAngle` | Optional override for the default `45` degree diagonal angle clamp. |
| `viewportViews` | Explicit pane assignments for fixed views, labels, cameras, or resolution scale. |
| `showLabels` | Renders DOM labels over panes. |
| `viewportLabels` | Label overrides or formatter. |
| `overlayOpacity` | Blend amount for overlay mode. |

Use this mental model when configuring panes: `layout` decides the arrangement, `paneCount` decides how many panes are visible, and `viewportViews` decides what each explicit pane renders. Simple pane assignments use the composed path; per-pane cameras and `resolutionScale` use the viewport/scissor path.

## `DebugView`

```ts
interface DebugView {
  id?: string
  label: string
  node?: DebugNode
  mode?: "passthrough" | "normal" | "depth" | "heatmap"
  source?: DebugViewSource
  scale?: number
  bias?: number
}
```

## `DebugViewportView`

```ts
interface DebugViewportView {
  view: number | DebugViewSource | DebugView
  label?: string
  camera?: Camera
  resolutionScale?: number
}
```

`resolutionScale` is normalized to `1`, `0.5`, or `0.25`.

## Built-in Sources

Built-in `DebugViewSource` values:

| Source | Notes |
| --- | --- |
| `beauty` | Final scene color. |
| `normal` | View-space geometry normals. |
| `depth` | View-space depth visualization. |
| `albedo` / `baseColor` | Base color without lighting. |
| `materialNormal` / `normalMap` | Material-perturbed normals. |
| `emissive` | Material emissive contribution. |
| `roughness` | Packed material scalar, defaults to `1` when unsupported. |
| `ao` | Packed material AO, defaults to `1` when unsupported. |
| `metallic` / `metalness` | Packed material scalar, defaults to `0` when unsupported. |
| `opacity` / `transparency` | Packed alpha scalar. |
| `wireframe` | Override-material pass. |
| `lightingOnly` | Neutral lighting override pass. |
| `reflectionOnly` | Reflective override pass. |
| `overdraw` | Heatmap-style overlap diagnostic with a separate legend ramp. |
| `shaderCost` | Shader-cost heatmap from shader-unit buckets. Click the viewport in the bundled overlay to sample a pixel. |

`shaderCost` is not a native GPU instruction counter. Render-pass timing and overlap diagnostics are separate signals, not ALU/TEX/occupancy data.

## Custom Views

Use `createCustomDebugView()` when a custom TSL node may be recreated by React but should keep a stable logical identity:

```ts
const fresnel = createCustomDebugView({
  id: "shader:fresnel",
  label: "Fresnel",
  node: fresnelNode,
  mode: "passthrough",
})
```

The compose runtime rebuilds when a custom node instance changes. The viewport render graph uses `id` in stable pass keys.

## Package Shape

The package has two public entrypoints:

| Entrypoint | Use it for |
| --- | --- |
| `threejs-debug-view` | Canonical runtime: planning, TSL compositor, `createDebugRenderPlan`, `createDebugPipelineRuntime`, `createDebugViewportRenderer`, layout/pane helpers, and `DebugViewsOptions` types. |
| `threejs-debug-view/r3f` | Thin adapter: `DebugViewLayer`, `DebugViews`, bundled Leva shell (`DebugViewLeva`, `mountDebugViewLeva`), and `useDebugViewsControls()`. |

See [Batteries Included](/threejs-debug-view/guides/batteries-included/) for what ships on npm versus what stays in the repo demo and docs site.

The repository demo and docs are separate from the package source:

- `components/debug-views/` contains package source.
- `src/` contains the Vite demo app.
- `packages/docs/` contains the Astro documentation site.

## Headless runtime (root export)

Import from `threejs-debug-view` when your app owns the WebGPU render loop:

- `createDebugRenderPlan()` — pass and pipeline view selection for a layout
- `createDebugPipelineRuntime()` — single-viewport `RenderPipeline` lifecycle
- `createDebugViewportRenderer()` — multi-pane scissor/viewport rendering
- `createDebugPipelineRuntimeKey()` — stable rebuild key for runtime caches
- `createDebugViewUniforms()` / `updateDebugViewUniforms()` — TSL uniform state
- `readHeatmapCostFromCanvas()` — shader-cost pixel sampling
- `DebugViewsOptions` / layout pane helpers from `debug-views-controls`

See [Headless Runtime](/threejs-debug-view/guides/headless-runtime/) for a full example.
