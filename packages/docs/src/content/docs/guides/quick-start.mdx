---
title: Quick Start
description: Add the debug view overlay and switch between built-in views.
---

Install the package with the R3F peers you already use in a Three.js React app.

```bash
pnpm add threejs-debug-view three react react-dom @react-three/fiber @react-three/drei leva
```

Use `threejs-debug-view/r3f` for React Three Fiber. The root package stays React-free.

```tsx
import { DebugViewLayer } from "threejs-debug-view/r3f"

function DebugLayer() {
  if (!import.meta.env.DEV) return null
  return <DebugViewLayer />
}
```

For app-owned controls, use the lower-level component and bundled hook from the same R3F entrypoint:

```tsx
import { DEFAULT_DEBUG_VIEWS, getDebugViewLabels } from "threejs-debug-view"
import { DebugViews, useDebugViewsControls } from "threejs-debug-view/r3f"

const controls = useDebugViewsControls({
  viewLabels: getDebugViewLabels(DEFAULT_DEBUG_VIEWS),
})

<DebugViews views={DEFAULT_DEBUG_VIEWS} {...controls} />
```

## View a specific source

If you want to pin one view from your own UI, pass `activeView` to `DebugViews`:

```tsx
const emissiveView = DEFAULT_DEBUG_VIEWS.findIndex((view) => view.source === "emissive")

<DebugViews
  views={DEFAULT_DEBUG_VIEWS}
  activeView={emissiveView}
  layout="single"
  showLabels
/>
```

The demo uses this same pattern for capture URLs such as `?debugView=emissive`. It also supports `?scene=overdraw` to open the foliage overdraw scene instead of the main helmet/material-sphere scene.

## Verification

Run the project checks before trusting a debug pipeline change:

```bash
pnpm typecheck
pnpm test
pnpm test:e2e
```

`pnpm test:e2e` exercises the local demo with Chromium and native WebGPU. On machines without native WebGPU it may skip locally, but CI treats that gate as a failure.
