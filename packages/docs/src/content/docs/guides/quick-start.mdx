Pick the path that matches your app. Both use the same WebGPU + TSL runtime; only the UI layer differs.

## Install without R3F

For a vanilla Three.js WebGPU app that owns its own render loop:

```bash
pnpm add threejs-debug-view three
```

Wire `createDebugPipelineRuntime` or `createDebugViewportRenderer` into your frame loop. No React, R3F, or Leva required.

```ts
import {
  DEFAULT_DEBUG_VIEWS,
  createDebugRenderPlan,
  createDebugPipelineRuntime,
  createDebugViewUniforms,
  resolveDebugViewLayout,
  updateDebugViewUniforms,
} from "threejs-debug-view"
import { WebGPURenderer } from "three/webgpu"

const views = DEFAULT_DEBUG_VIEWS
const layout = resolveDebugViewLayout("single")
const plan = createDebugRenderPlan(views, 0, layout)
const uniforms = createDebugViewUniforms()

const runtime = createDebugPipelineRuntime(
  scene,
  camera,
  plan,
  layout,
  renderer as WebGPURenderer,
  uniforms,
)

function animate() {
  updateDebugViewUniforms(uniforms, plan.activePipelineView, layout, plan.pipelineViews.length, 1)
  runtime.pipeline.render()
  requestAnimationFrame(animate)
}
```

See [Headless Runtime](/threejs-debug-view/guides/headless-runtime/) for multi-pane layouts, viewport scaling, and when to use `createDebugViewportRenderer`.

## Install with R3F

For a React Three Fiber app, add the adapter peers:

```bash
pnpm add threejs-debug-view three react react-dom @react-three/fiber @react-three/drei leva
```

## Batteries included (R3F)

Use `threejs-debug-view/r3f` when you want the default overlay with Leva controls and built-in views already wired.

```tsx
import { DebugViewLayer } from "threejs-debug-view/r3f"

function DebugLayer() {
  if (!import.meta.env.DEV) return null
  return <DebugViewLayer />
}
```

Mount `DebugLayer` inside your `<Canvas>`. Leva mounts on `document.body` automatically — you do not import or place `<Leva />` yourself unless you set `showLeva={false}` and use `DebugViewLeva` for custom shell placement.

See [Batteries Included](/threejs-debug-view/guides/batteries-included/) for what ships out of the box, custom Leva placement, and the React-free core export.

## Bring your own controls (R3F)

Use the lower-level component and bundled hook when your app owns UI state:

```tsx
import { DEFAULT_DEBUG_VIEWS, getDebugViewLabels } from "threejs-debug-view"
import { DebugViews, useDebugViewsControls } from "threejs-debug-view/r3f"

const controls = useDebugViewsControls({
  viewLabels: getDebugViewLabels(DEFAULT_DEBUG_VIEWS),
})

<DebugViews views={DEFAULT_DEBUG_VIEWS} {...controls} />
```

## View a specific source

Pin one view from your own UI by passing `activeView` to `DebugViews`:

```tsx
const emissiveView = DEFAULT_DEBUG_VIEWS.findIndex((view) => view.source === "emissive")

<DebugViews
  views={DEFAULT_DEBUG_VIEWS}
  activeView={emissiveView}
  layout="single"
  showLabels
/>
```

The demo uses this pattern for capture URLs such as `?debugView=emissive`. It also supports `?scene=overdraw` to open the foliage overdraw scene instead of the main helmet/material-sphere scene.

For a fixed breakdown presentation shot (Normal, Shader Complexity, Albedo, Depth with labels and legends, Leva hidden), open `/?capture=social&scene=overdraw`. From the repository root, run `pnpm capture:social` while the Vite dev server is running to write `capture/breakdown-social-x.png`.

## Verification

Run the project checks before trusting a debug pipeline change:

```bash
pnpm typecheck
pnpm test
pnpm test:e2e
```

`pnpm test:e2e` exercises the local demo with Chromium and native WebGPU. On machines without native WebGPU it may skip locally, but CI treats that gate as a failure.
