`threejs-debug-view` exposes named debug views for the buffers your scene already produces: beauty, normals, depth, material channels, wireframe, lighting-only, reflection-only, overlap, and shader cost.

## Without R3F

Import from **`threejs-debug-view`** when you own the WebGPU render loop. You get view definitions, render planning, and runtime helpers such as `createDebugPipelineRuntime` and `createDebugViewportRenderer`. No React, R3F, or Leva required.

[Install without R3F](/threejs-debug-view/guides/quick-start/#install-without-r3f) · [Headless Runtime](/threejs-debug-view/guides/headless-runtime/)

## With R3F

Import from **`threejs-debug-view/r3f`** for a batteries-included overlay: `DebugViewLayer`, Leva controls, and DOM labels/legends inside a `<Canvas>`.

```tsx
import { DebugViewLayer } from "threejs-debug-view/r3f"

function DebugLayer() {
  if (!import.meta.env.DEV) return null
  return <DebugViewLayer />
}
```

[Install with R3F](/threejs-debug-view/guides/quick-start/#install-with-r3f) · [Batteries Included](/threejs-debug-view/guides/batteries-included/)

Keep debug views behind a dev flag unless you intentionally expose them in production.

## What it does

- **Built-in sources** — beauty, normals, depth, albedo, roughness, AO, opacity, emissive, wireframe, lighting-only, reflection-only, overlap, and shader cost.
- **Render layouts** — single, overlay, split, diagonal breakdown, row, column, and grid.
- **Explicit pane assignments** — fixed views, labels, per-pane cameras, and resolution scaling.
- **Headless runtime** — same WebGPU pipeline without React.
- **Custom TSL views** — add shader-specific inspection without forking the built-in list.
